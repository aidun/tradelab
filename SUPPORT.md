# Support

## What this document is for

This document explains where to ask for help depending on what you need.

## Use GitHub issues for

- reproducible bugs
- feature requests
- documentation improvements
- roadmap-aligned product discussion

Please use the provided issue templates when possible.

## Do not use public issues for

- sensitive security reports
- secrets or credential problems
- private operational details that should not be public

For those cases, follow [SECURITY.md](SECURITY.md).

## Before opening a support request

Please check:

- [README.md](README.md)
- [docs/user-guide.md](docs/user-guide.md)
- [docs/deployment.md](docs/deployment.md)
- [docs/developer-guide.md](docs/developer-guide.md)

## Maintainer support boundaries

The maintainers aim to keep the repository clear and responsive, but support is best-effort.

Priority is typically given to:

- issues aligned with the roadmap
- reproducible defects
- documentation gaps that block adoption
- release, deployment, or CI problems affecting the mainline workflow
